## 1. 🔄 Полная замена движка генерации (`src/generator.py`)

### Было (v1 — `208c7c3`)
- Использовался **xlsxwriter** — библиотека, которая создаёт файлы «с нуля»
- Шаблон копировался через `shutil.copyfile`, затем поверх него создавался новый файл
- Стили задавались вручную через `add_format()` — при малейшем изменении шаблона код ломался
- Разметка страницы (`setup_page_layout`) была захардкожена в Python-коде

### Стало (v2 — текущая)
- Переход на **openpyxl** с загрузкой шаблона через `load_workbook(template_path)`
- **Шаблон полностью сохраняется**: все стили, ширина колонок, высота ячеек, шрифты — остаются из оригинального XLSX
- Запись данных через `cell.value = val` без пересоздания форматирования
- Добавлен **сборщик мусора** (`gc.collect()`) для освобождения памяти после генерации каждого диплома

## 2. 📊 Улучшенный парсер Excel (`src/parser.py`)

### Было (v1)
- Парсинг только одной строки заголовков (row 1)
- Шаг поиска колонок был фиксированным (`col += 4`)
- Метаданные листа (специальность, квалификация, годы) **не извлекались**
- Отсутствовала фильтрация служебных строк (итого, руководитель и т.д.)

### Стало (v2)
- **Двухуровневый парсинг заголовков**: row 1 (общеобразовательные) + row 2 (детальные ОН/БМ) с приоритетом row 2
- **Динамический поиск** столбцов — без жёсткого шага, каждый столбец проверяется индивидуально
- Новая функция `_parse_sheet_meta()` — извлекает:
  - Специальность (KZ/RU)
  - Квалификацию (KZ/RU)
  - Год поступления / выпуска
- **Фильтрация** нестудентских строк по маркерам: `сағат саны`, `итого`, `руководитель`, `директор` и др.
- Числовая проверка индекса — нечисловые строки (заголовки, подписи) автоматически пропускаются
- **Системный fallback** для определения типа предмета (зачёт vs оценка) на основе наличия часов и кредитов

## 3. 🔧 Исправления в утилитах (`src/utils.py`)

### Регулярные выражения для парсинга часов/кредитов
- **Было:** `r"(\d+)с"` — не допускал пробелы между числом и буквой
- **Стало:** `r"(\d+)\s*с"` — корректно парсит `72с`, `72 с`, `72  с`
- **Было:** `r"(\d+)с-(\d+(?:[.,]\d+)?)к"` — жёсткий формат без пробелов
- **Стало:** `r"(\d+)\s*с\s*-\s*(\d+(?:[.,]\d+)?)\s*к"` — допускает пробелы во всех позициях

## 4. 🎯 Интеллектуальный маппинг оценок

### Новые возможности в генераторе
| Функция | Описание |
|---------|----------|
| **Prefix-based lookup** | Поиск оценок по префиксам модулей (БМ, КМ, ПМ, ОН, РО) |
| **Cross-language fallback** | Автозамена ОН ↔ РО, КМ ↔ ПМ при отсутствии совпадения |
| **Агрегация часов** | Автосуммирование часов подмодулей для заголовков модулей (КМ, ПМ) |
| **Типизация предметов** | Корректное разделение на практики, факультативы и обычные дисциплины |
| **Практики с оценкой** | Кәсіптік/профессиональная практика теперь получает оценку, а не «зачтено» |

## 7. 📝 Управление памятью

- **Было:** файл просто закрывался через `self.workbook.close()`
- **Стало:**
  ```python
  self.workbook.save(self.output_path)
  self.workbook.close()
  self.workbook = None
  gc.collect()
  ```
  Явное освобождение ссылки и сборка мусора — критично при пакетной генерации десятков дипломов.

## Резюме

Проект прошёл через **масштабный рефакторинг** за 2 недели:

1. **Надёжность** — переход от «создания с нуля» к «заполнению шаблона» исключил ошибки форматирования
2. **Точность** — многоуровневый парсинг и cross-language fallback гарантируют правильное сопоставление оценок
3. **Производительность** — управление памятью через `gc.collect()` для пакетной обработки
4. **Тестируемость** — 50+ тестовых и отладочных скриптов для верификации каждого компонента
5. **Гибкость** — regex-паттерны допускают вариации форматов из Excel
