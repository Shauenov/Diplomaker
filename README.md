## 1. Обзор проекта

### 1.1 Что это?

**Diploma Automation System** — система автоматической генерации приложений к дипломам (diploma supplements) для студентов колледжа. Система считывает оценки студентов из Excel-файла и формирует готовые к печати Excel-файлы приложений к дипломам.

### 1.2 Поддерживаемые программы

| # | Специальность | Код | Группы | Листы Excel |
|---|--------------|-----|--------|-------------|
| 1 | **IT (Информационные технологии)** | 3F | 3Ғ-1, 3Ғ-2, 3Ғ-3, 3Ғ-4 | Листы начинающиеся на `3Ғ` |
| 2 | **Бухгалтерский учёт (Accounting)** | 3D | 3D-1, 3D-2 | Листы начинающиеся на `3D` |

### 1.3 Поддерживаемые языки

| Код | Язык | Применение |
|-----|------|------------|
| **KZ** | Қазақша (Казахский) | Казахскоязычные дипломы |
| **RU** | Русский | Русскоязычные дипломы |

Для каждого студента система генерирует **2 диплома** — на казахском и русском языках.

### 1.4 Текущий статус

- ✅ Phase 1: Foundation (Core модели, конфигурация, утилиты)
- ✅ Phase 2: Data Layer (Excel парсер и генератор)
- ✅ Phase 3: Testing (Юнит-тесты и интеграционное тестирование)
- ✅ Phase 4: Production deployment (завершено)

> **Статус:** Разработка основного функционала успешно завершена. Проект готов к полноценному использованию в production среде.

## 2. Назначение и бизнес-логика

### 2.1 Исходные данные

Система работает с Excel-файлом формата:
```
2025-2026 диплом бағалары (ТОЛЫҚ) қызыл диплом жазылған соңғысы точно (1).xlsx
```

Этот файл содержит **все оценки всех студентов** за учебный год, организованные по листам (группам).

### 2.2 Структура исходного Excel

```
Excel-файл
├── Лист "3Ғ-1" (IT группа 1)
├── Лист "3Ғ-2" (IT группа 2)
├── Лист "3Ғ-3" (IT группа 3)
├── Лист "3Ғ-4" (IT группа 4)
├── Лист "3D-1" (Бухгалтеры группа 1)
└── Лист "3D-2" (Бухгалтеры группа 2)
```

**Структура каждого листа:**

| Строка | Содержание |
|--------|------------|
| Строка 1 (инд. 0) | Порядковые номера предметов |
| Строка 2 (инд. 1) | Названия предметов (двуязычные: KZ\nRU) |
| Строка 3 (инд. 2) | Подзаголовки модулей |
| Строка 4 (инд. 3) | Часы и кредиты (`72с-3к`) |
| Строка 5 (инд. 4) | Метки колонок: `п`, `б`, `цэ`, `трад` |
| Строка 6+ (инд. 5+) | **Данные студентов** |

**Колонки для каждого предмета (4 колонки):**

| Колонка | Тип | Описание |
|---------|-----|----------|
| `п` | Процент (0–100) | Балл в процентах |
| `б` | GPA (0.0–4.0) | Средний балл |
| `цэ` | Буква (A, B+, C) | Буквенная оценка |
| `трад` | Текст | Традиционная оценка (жақсы / хорошо) |

### 2.3 Что генерируется

Для **каждого студента** создаются два Excel-файла (KZ и RU), содержащие **4 страницы** приложения к диплому:

| Стр. | Содержание |
|------|------------|
| 1 | Общеобразовательные предметы (13 предметов) |
| 2 | Базовые модули (БМ) + Профессиональные модули КМ 01–КМ 04 |
| 3 | Профессиональные модули КМ 05–КМ 09 |
| 4 | КМ 10 + Практика + Аттестация + Факультативы |

### 5.2 `core/converters.py` — Конвертация оценок (201 строка)

Модуль конвертации процентных баллов в все форматы оценок.

#### Основная функция: `convert_score_to_grade(points) → Grade`

Принимает строку с процентной оценкой, возвращает объект `Grade` со всеми полями.

```python
grade = convert_score_to_grade("85")
# grade.letter == "B+"
# grade.gpa == 3.33
# grade.traditional_kz == "4 (жақсы)"
# grade.traditional_ru == "4 (хорошо)"
```

**Обработка граничных случаев:**
- `None` / `""` / `"nan"` → пустой Grade
- Невалидный формат → `ValidationError`
- Вне диапазона 0–100 → `ValidationError`

#### Вспомогательные функции (шорткаты)

| Функция | Возвращает | Пример |
|---------|-----------|--------|
| `get_gpa_value("85")` | `3.33` | Только GPA |
| `get_letter_grade("85")` | `"B+"` | Только буква |
| `get_traditional_grade("85", "KZ")` | `"4 (жақсы)"` | Только трад. оценка |

#### Внутренний механизм: `_lookup_grade_threshold(score)`

Использует таблицу `GRADE_THRESHOLDS` из `config/settings.py`. Пороги отсортированы по убыванию, первое совпадение (`score >= threshold`) определяет оценку.

### 5.3 `core/utils.py` — Утилитарные функции (264 строки)

| Функция | Описание | Пример |
|---------|----------|--------|
| `is_module_header(name)` | Является ли предмет заголовком модуля (КМ, БМ, ПМ) | `is_module_header("КМ 01 Web")` → `True` |
| `normalize_key(text)` | Нормализация для нечёткого поиска | `normalize_key("КМ 01 Web")` → `"км1web"` |
| `clean_subject_name(text)` | Разделить двуязычное название | `"KZ\nRU"` → `("KZ", "RU")` |
| `parse_hours_credits(text)` | Извлечь часы и кредиты | `"72с-3к"` → `("72", "3")` |
| `robust_clean(val)` | Очистка значения ячейки | `NaN` → `""`, `85` → `"85"` |
| `format_float_value(val, dp)` | Форматирование float | `3.33` → `"3.33"` |

#### Детали `normalize_key()`

Алгоритм нормализации:
1. Перевод в нижний регистр
2. Удаление знаков препинания (`.`, `,`, `:`)
3. Удаление всех пробелов
4. Удаление ведущих нулей в номерах (`км01` → `км1`)

Это позволяет сопоставлять предметы из шаблона с Excel-данными, даже если написание немного отличается.

#### Детали `is_module_header()`

Определяет заголовки модулей по префиксам и ключевым словам:
- Префиксы: `КМ`, `БМ`, `ПМ`, `СМ`
- Ключевые слова: `практика`, `аттеста`, `зачет`, `зачёт`

## 8. Пакет `data/` — Слой данных

### 8.1 `data/excel_parser.py` — Парсер Excel (439 строк)

#### Класс `ExcelParser`

**Назначение:** Чтение Excel-файла с оценками студентов и преобразование в объекты `Student`.

**Инициализация:**

```python
parser = ExcelParser(
    source_file="grades.xlsx",
    sheet_names=["3Ғ-1", "3Ғ-2"],   # None = авто-определение
    row_subject_names=1,              # Данные о предметах — строка 2
    row_hours_credits=3,              # Часы/кредиты — строка 4
    row_data_start=4,                 # Первый студент — строка 5
    col_student_name=1,               # ФИО — колонка B
    col_start_subjects=2,             # Первый предмет — колонка C
    cols_per_subject=4,               # 4 колонки на предмет
)
```

**Основные методы:**

| Метод | Описание | Возвращает |
|-------|----------|-----------|
| `parse(sheet_name)` | Парсинг одного листа | `List[Student]` |
| `parse_all_sheets()` | Парсинг всех листов | `Dict[str, List[Student]]` |
| `get_subjects(sheet_name)` | Только предметы с часами | `List[Subject]` |
| `validate_excel_structure(file)` | Статическая валидация | `bool` |

**Внутренний процесс парсинга:**

```
1. _load_dataframe(sheet)     → pd.DataFrame (raw, без заголовков)
2. _parse_subjects(df)        → List[Subject] (из строки 1)
3. _parse_hours_credits(df)   → Dict[col_idx, (hours, credits)]
4. _parse_students(df, subj)  → List[Student] (из строк 4+)
```

**Алгоритм _parse_subjects:**
1. Начиная с колонки 2, читает каждую 4-ю колонку (шаг = `cols_per_subject`)
2. Пропускает пустые ячейки
3. Разделяет двуязычное название через `clean_subject_name()`
4. Определяет, является ли предмет заголовком модуля

**Алгоритм _parse_students:**
1. Итерирует строки начиная с `row_data_start`
2. Пропускает пустые строки
3. Для каждого предмета читает 4 ячейки: points, letter, gpa, traditional
4. Создаёт объект `Grade` и добавляет к `Student.grades`

### 8.2 `data/excel_generator.py` — Генератор дипломов (654 строки)

#### Класс `DiplomaGenerator`

**Назначение:** Создание Excel-файла диплома на основе данных студента.

**Инициализация:**

```python
gen = DiplomaGenerator(
    program=Program.IT,
    language=Language.KZ,
    academic_year="2025-2026",

```

**Основные методы:**

| Метод | Описание | Возвращает |
|-------|----------|-----------|
| `generate(student, subjects)` | Генерация в памяти | `bytes` (Excel) |
| `generate_to_file(student, subjects, path)` | Генерация в файл | `str` (путь) |

**Структура выходного файла:**

Каждый диплом — Excel-книга с 4 листами (страницами):

```
Бет 1 / Лист 1 — Заголовок (ФИО, номер, годы) + Общеобразовательные предметы
Бет 2 / Лист 2 — БМ + КМ 01–04
Бет 3 / Лист 3 — КМ 05–09
Бет 4 / Лист 4 — КМ 10 + Практика + Аттестация + Факультативы
```

**Формат таблицы предметов (8 колонок):**

| Колонка | Содержание | Ширина |
|---------|-----------|--------|
| A | №п/п | 4 |
| B | Название предмета | 30 |
| C | Часы | 5 |
| D | Кредиты | 4 |
| E | Баллы (%) | 5 |
| F | Буквенная оценка | 4 |
| G | GPA | 5 |
| H | Традиционная оценка | 13 |

**Типы предметов и их отображение:**

| Тип | №п/п | Часы/Кредиты | Оценки | Стиль |
|-----|------|-------------|--------|-------|
| Обычный предмет | ✓ | ✓ | ✓ (все 4 поля) | Обычный |
| Заголовок модуля (КМ/ПМ) | ✓ | Σ (сумма подчинённых) | — | **Жирный** |
| Факультатив | ✓ | ✓ | Трад.= «сынақ/зачтено» | Обычный |

**Механизм сопоставления предметов (_lookup_grade):**

1. Прямой поиск по имени в `grades_data`
2. Нормализованный поиск через `normalize_key()`
3. Нечёткий поиск для практики и аттестации

**Списки предметов (hardcoded):**

Генератор содержит встроенные списки предметов для IT программы на обоих языках (`_PAGE1_KZ`, `_PAGE2_KZ`, ..., `_PAGE4_RU`). Эти списки определяют порядок и названия предметов в дипломе.

## 16. Потоки данных (Data Flow)

### 16.1 Основной поток (Новая архитектура)

```
┌──────────────────────┐
│ Excel Source File     │    2025-2026 диплом бағалары...xlsx
│ (Оценки студентов)   │
└──────────┬───────────┘
           │
           v
┌──────────────────────┐
│ data/excel_parser.py │    ExcelParser.parse(sheet_name)
│   _parse_subjects()  │    → List[Subject]
│   _parse_students()  │    → List[Student] (с Grade для каждого предмета)
└──────────┬───────────┘
           │
           v
┌──────────────────────┐
│  core/converters.py  │    convert_score_to_grade(points)
│  (если нужна деривация) → Grade (letter, gpa, traditional_kz, traditional_ru)
└──────────┬───────────┘
           │
           v
┌──────────────────────┐
│ data/excel_generator │    DiplomaGenerator.generate(student, subjects)
│  _build_grades_data  │    → Сопоставление предмет → оценки
│  _write_subjects     │    → Запись таблиц в Excel
│  _write_header       │    → Запись заголовка (ФИО, номер, годы)
└──────────┬───────────┘
           │
           v
┌──────────────────────┐
│ Output Excel Files   │    {ФИО}_KZ_2025-2026.xlsx
│ (Готовые дипломы)    │    {ФИО}_RU_2025-2026.xlsx
└──────────────────────┘
```

### 16.2 Легаси поток (main.py)

```
Excel File
    │
    v
pd.ExcelFile(source) → df = xl.parse(sheet_name)
    │
    v
src/parser.py: parse_excel_sheet(df, sheet_name) → List[Dict]
    │
    v
configs/__init__.py: get_config(group, lang) → (config, terms, template)
    │
    v
src/generator.py: DiplomaGenerator(template, output, config, terms)
    │                .fill_student_data(student_dict)
    │                .close()
    v
Output Excel Files
```

### 16.3 Сопоставление предметов (Subject Matching)

Критически важный этап — сопоставление предметов из шаблона с данными из Excel:

```
Шаблон: "КМ 03 Front-end web ресурстарды құру"
                    │
                    v  normalize_key()
                "км3front-endwebресурстардықұру"
                    │
                    v  Поиск в grades_data["__norm__"]
Excel:  "км3front-endwebресурстардықұру" → ✓ Найдено!
```

Fallback при ненахождении:
1. Прямой поиск по полному имени
2. Нормализованный поиск
3. Нечёткий поиск по ключевым словам (практика, аттестация)

## 17. Система оценок

### 17.1 Шкала оценок (полная таблица)

| Баллы (%) | Буква | GPA | Трад. (KZ) | Трад. (RU) | Цифра |
|-----------|-------|-----|------------|------------|-------|
| 95–100 | A | 4.0 | өте жақсы | отлично | 5 |
| 90–94 | A- | 3.67 | өте жақсы | отлично | 5 |
| 85–89 | B+ | 3.33 | жақсы | хорошо | 4 |
| 80–84 | B | 3.0 | жақсы | хорошо | 4 |
| 75–79 | B- | 2.67 | жақсы | хорошо | 4 |
| 70–74 | C+ | 2.33 | жақсы | хорошо | 4 |
| 65–69 | C | 2.0 | қанағат | удовл | 3 |
| 60–64 | C- | 1.67 | қанағат | удовл | 3 |
| 55–59 | D+ | 1.33 | қанағат | удовл | 3 |
| 50–54 | D | 1.0 | қанағат | удовл | 3 |
| 0–49 | F | 0.0 | қанағаттанарлықсыз | неуд | 2 |

### 17.2 Деривация оценок

Если в Excel отсутствует колонка `трад` (некоторые предметы имеют только 3 подколонки), система **выводит** традиционную оценку из процентных баллов:

```python
# Упрощённая деривация из TZ:
if score >= 95: "5 (өте жақсы)"
elif score >= 75: "4 (жақсы)"
elif score >= 50: "3 (қанағат)"
else: "2 (қанағатсыз)"
```

### 17.3 Специальные случаи

| Случай | Поведение |
|--------|----------|
| Пустая ячейка / NaN | Пустая строка |
| `#REF!` (ошибка Excel) | Пустая строка |
| `0` | Пустая строка (≠ оценка 0) |
| Факультативы | `"сынақ"` (KZ) / `"зачтено"` (RU) |
| Заголовки модулей | Без оценок, сумма часов подчинённых |

## 18. Обработка ошибок

### 18.1 Иерархия исключений

```python
try:
    parser = ExcelParser(source_file="missing.xlsx")
except ConfigurationError:
    # Файл не найден

try:
    students = parser.parse("NonexistentSheet")
except ParseError:
    # Ошибка чтения/парсинга Excel

try:
    grade = convert_score_to_grade("abc")
except ValidationError:
    # Невалидный формат оценки

try:
    gen.generate_to_file(student, subjects, "output.xlsx")
except GenerationError:
    # Ошибка записи файла
```

### 18.2 Стратегии обработки

| Уровень | Стратегия |
|---------|----------|
| Ячейка Excel | `robust_clean()` — безопасная очистка |
| Оценка студента | `convert_score_to_grade()` — пустой Grade при пустом значении |
| Студент целиком | Пропуск + запись в `ProcessingResult.errors` |
| Лист Excel | Продолжение со следующим листом |
| Файл Excel | Ошибка на уровне BatchProcessor |

### 18.3 Логирование

- Основной лог: `diploma_automation.log`
- Отладочный лог: `debug.log`
- Уровень: `INFO` (продакшн), `DEBUG` (при `DEBUG_MODE=True`)

## 21. Известные ограничения и TODO

### 21.1 Дублирование кода

Система содержит **параллельные реализации** одной и той же логики:
- `core/utils.py` vs `src/utils.py` — утилитарные функции
- `core/converters.py` vs `src/utils.py` (calc_letter_grade) — конвертация
- `data/excel_parser.py` vs `src/parser.py` — парсинг
- `data/excel_generator.py` vs `src/generator.py` — генерация
- `config/programs.py` vs `configs/subjects_*.py` — списки предметов

> **Рекомендация:** Мигрировать `main.py` на новую архитектуру и удалить `src/`.

### 21.2 Незавершённые компоненты

- **Бухгалтерия (PROGRAM_ACCOUNTING):** Списки предметов в `config/programs.py` частично пусты (TODO)
- **Word-шаблоны (.docx):** Система поддерживает только Excel. TZ описывает DOCX-формат с зелёными плейсхолдерами — не реализовано
- **PDF-генерация:** Не реализована
- **Красный диплом:** Логика определения не реализована

### 21.3 Различия в нормализации

Два `normalize_key()`:
- `core/utils.py` — сохраняет казахские буквы
- `src/utils.py` — **заменяет** казахские буквы на русские (ғ→г, қ→к, ү→у...)

Это может привести к несовпадениям при перекрёстном использовании.

### 21.4 Различия в пороговых значениях

- `core/converters.py` / `config/settings.py` — 70-74 = `"4 (жақсы)"` (хорошо)
- `src/utils.py` `calc_traditional_grade()` — 70–89 = `"жақсы"` (без цифры)

Форматы традиционных оценок различаются между реализациями.

### 21.5 Hardcoded пути

- `config/settings.py` содержит абсолютные пути к Excel-файлу
- Привязка к конкретному пользователю и каталогу
